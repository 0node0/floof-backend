# Floof v4 — Railway Backend Deployment Guide

Follow these steps to deploy `floof-backend` to Railway.

---

## 1. Create Railway Project

1. Go to [railway.com/dashboard](https://railway.com/dashboard)
2. Click **"New Project"**
3. Select **"Deploy from GitHub repo"**
4. Choose `0node0/floof-backend`
5. Railway will auto-detect the `Dockerfile` and `railway.toml`

---

## 2. Add Environment Variables

In your Railway project → Variables, add these:

| Variable | Value / Where to get it |
|----------|------------------------|
| `NODE_ENV` | `production` |
| `PORT` | `9000` (Railway will override with its own, but good fallback) |
| `API_TYPE` | `seller` |
| `API_BASE_URL` | `https://api.floof.space` (your custom domain) |
| `ALLOWED_ORIGINS` | `https://floof.space` |
| `STRIPE_API_KEY` | `sk_live_...` or `sk_test_...` from Stripe Dashboard → Developers → API keys |
| `STRIPE_WEBHOOK_SECRET` | `whsec_...` — create an endpoint in Stripe Dashboard → Webhooks pointing to `https://api.floof.space/webhooks/stripe` |
| `PRINTFUL_API_KEY` | Your Printful API key |
| `PRINTFUL_WEBHOOK_SECRET` | Any random string (e.g., `floof_webhook_2026_$(openssl rand -hex 16)`) |
| `AUTH_CORS` | `https://floof.space` |
| `STORE_CORS` | `https://floof.space` |
| `JWT_SECRET` | `$(openssl rand -hex 32)` |
| `COOKIE_SECRET` | `$(openssl rand -hex 32)` |

> **Note:** `DB_URL` and `REDIS_URL` will be auto-generated by Railway when you add Postgres and Redis services (next step).

---

## 3. Add Database Services

1. In your Railway project, click **"New"** → **"Database"** → **"Add PostgreSQL"**
2. Railway creates it and sets `PGHOST`, `PGPORT`, `PGUSER`, `PGPASSWORD`, `PGDATABASE` automatically
3. Click **"New"** → **"Add Redis"**
4. Railway creates it and sets `REDISHOST`, `REDISPORT`, `REDISUSER`, `REDISPASSWORD` automatically

These are automatically wired into your `Dockerfile` via Railway's environment injection.

---

## 4. Deploy & Verify

1. Railway will auto-deploy on push. Since you've already pushed, it may have started deploying.
2. Check the **Deploy Logs** to ensure the build succeeds.
3. Once deployed, Railway gives you a temporary URL (e.g., `https://floof-backend.up.railway.app`)

### Set Custom Domain
1. In Railway → Settings → Domain, click **"Custom Domain"**
2. Add `api.floof.space`
3. Railway gives you a CNAME target (e.g., `railway.app` or a specific subdomain)
4. Go to **Spaceship** → DNS settings for `floof.space` → Add CNAME record:
   - Name: `api`
   - Value: The Railway CNAME target

---

## 5. Run Migrations

After first deploy, you need to run database migrations:

**Method A: Railway CLI (if installed)**
```bash
railway login
railway link  # select your project
cd /Users/chrisslowik/Projects/floof-backend
railway run npx medusa db:migrate
```

**Method B: Railway Dashboard → "New" → "Run Command"**
```
npx medusa db:migrate
```

Then seed:
```
npx medusa exec dist/scripts/seed.js
```

Or use the `start.sh` script which handles this automatically:
```bash
docker run --rm -e DB_URL=$DB_URL -e REDIS_URL=$REDIS_URL ghcr.io/0node0/floof-backend:latest npx medusa db:migrate
```

---

## 6. Get Your Publishable API Key

After seeding, check the Railway logs for:
```
Created publishable API key: pk_xxxxxxxx
```

Or query it via the Admin API:
```bash
curl https://api.floof.space/app/admin/api-keys \
  -H "Authorization: Bearer <your-admin-token>"
```

You'll need this `pk_...` value for the frontend's `PUBLIC_MEDUSA_PK` environment variable.

---

## 7. Configure Webhooks

### Stripe
1. Stripe Dashboard → Developers → Webhooks
2. Add endpoint: `https://api.floof.space/webhooks/stripe`
3. Select events:
   - `payment_intent.succeeded`
   - `payment_intent.payment_failed`
4. Copy the **Signing secret** (`whsec_...`) to Railway `STRIPE_WEBHOOK_SECRET`

### Printful
1. Printful Dashboard → Stores → Your Store → Settings → Webhooks
2. Add webhook URL: `https://api.floof.space/webhooks/printful`
3. Use the same secret you set in `PRINTFUL_WEBHOOK_SECRET`

---

## 8. Verify Health Check

```bash
curl https://api.floof.space/health
curl https://api.floof.space/health/rate-limit
curl https://api.floof.space/health/webhooks
```

All should return `200 OK`.

---

## Troubleshooting

| Issue | Fix |
|-------|-----|
| Build fails with "Dockerfile not found" | Ensure `Dockerfile` is in repo root, not a subdirectory |
| Migrations fail | Check `DB_URL` is correctly set and Postgres service is running |
| 500 on `/webhooks/stripe` | Verify `STRIPE_WEBHOOK_SECRET` matches Stripe Dashboard |
| Products not showing | Run seed, then add products in Printful and trigger a sync |
